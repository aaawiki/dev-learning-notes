# 📖 日常使用指南

## 一、用什么写笔记？

**Markdown 文件 + VS Code**

| 对比项 | Markdown | Word/文档 | Notion/语雀 |
|--------|----------|-----------|-------------|
| GitHub 渲染 | ✅ 原生支持 | ❌ 不显示 | ❌ 外部链接 |
| 版本追踪 | ✅ 每行改动可见 | ❌ 二进制文件 | ❌ 无法追踪 |
| 离线使用 | ✅ 随时写 | ✅ | ❌ 需联网 |
| 学习成本 | ⭐ 5分钟学会 | - | - |

**推荐编辑器**：VS Code（你应该已经装了）
- 左边写 Markdown，右边按 `Ctrl + Shift + V` 预览效果
- 不需要装任何插件就能写

## 二、项目结构说明

```
dev-learning-notes/
├── README.md            ← 项目说明（不用动）
├── 笔记模板.md           ← 每次写笔记，复制这个模板
├── notes/               ← 【日志笔记】按日期记录
│   └── 2026-07/         ← 按年月分文件夹
│       └── 2026-07-15-标题.md
├── topics/              ← 【主题笔记】按主题深度总结
│   └── 例如-Python异步编程.md
└── assets/              ← 截图、图片放这里
```

**两种笔记什么时候用？**
- **日志笔记** (notes/)：今天遇到一个 bug，解决了，随手记一篇。低门槛，记下来就行。
- **主题笔记** (topics/)：学了很久的某个主题，系统整理一篇。比如「Python 装饰器完全指南」。

## 三、写一篇新笔记的完整流程

### 第 1 步：复制模板

打开 `笔记模板.md`，全选复制内容。

### 第 2 步：新建文件

在 `notes/2026-07/` 文件夹下新建文件，命名规则：

```
年-月-日-简短标题.md
```

例如：`2026-07-15-Python列表推导式踩坑.md`

### 第 3 步：填写内容

粘贴模板，填写：
- **问题描述**：遇到了什么问题？贴上报错信息和代码
- **解决方法**：怎么解决的？分步骤写
- **原理分析**：为什么会这样？
- **感悟**：学到了什么？

### 第 4 步：上传到 GitHub

打开终端（VS Code 里按 `` Ctrl + ` `` 打开终端），执行：

```bash
# 进入项目目录（只需第一次执行，之后终端没关就不用重复）
cd D:\dev-learning-notes

# 添加所有改动
git add .

# 提交，写一句话说明今天记了什么
git commit -m "记录：Python列表推导式踩坑"

# 推送到 GitHub
git push
```

完成！去 https://github.com/aaawiki/dev-learning-notes 就能看到你的笔记了。

## 四、常用场景速查

### 场景 1：今天写了一篇笔记

```bash
cd D:\dev-learning-notes
git add .
git commit -m "记录：今天解决的问题标题"
git push
```

### 场景 2：改了之前的笔记

```bash
cd D:\dev-learning-notes
git add .
git commit -m "更新：修正了XXX笔记的内容"
git push
```

### 场景 3：在另一台电脑上想继续写

```bash
# 第一次：克隆仓库到本地
git clone https://github.com/aaawiki/dev-learning-notes.git

# 之后每次开始前：拉取最新内容
cd dev-learning-notes
git pull

# 写完笔记后：正常提交推送
git add .
git commit -m "记录：新笔记标题"
git push
```

### 场景 4：笔记里要插图片

1. 把图片放到 `assets/` 文件夹
2. 在笔记里这样引用：

```markdown
![图片说明](../assets/screenshot.png)
```

### 场景 5：想看自己一共写了多少笔记

```bash
# 查看所有提交记录
git log --oneline

# 统计笔记数量
find notes/ -name "*.md" | wc -l
```

## 五、Git 命令速查卡

| 命令 | 作用 | 什么时候用 |
|------|------|-----------|
| `git add .` | 添加所有改动 | 每次写完笔记 |
| `git commit -m "说明"` | 提交到本地 | 紧接着上一步 |
| `git push` | 推送到 GitHub | 紧接着上一步 |
| `git pull` | 拉取远程更新 | 换电脑时、开始写之前 |
| `git status` | 查看当前状态 | 不确定有没有改动时 |
| `git log --oneline` | 查看提交历史 | 想看写了多少笔记 |

**日常只需要记住三个命令**：`git add .` → `git commit -m "说明"` → `git push`

## 七、好习惯建议

1. **随手记**：解决完问题立刻记，不要等「以后再补」
2. **贴代码**：把报错信息和代码贴进去，未来的你会感谢现在的你
3. **写感悟**：解决方法网上都有，但你的感悟是独一无二的
4. **定期整理**：每月把日志笔记中同类问题整理成一篇主题笔记
5. **标题清晰**：标题写清楚解决的是什么问题，方便以后搜索
