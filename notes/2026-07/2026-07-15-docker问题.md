# 笔记标题：一句话说清楚这篇笔记讲什么

> 日期：2026-07-16

---

## 问题




### 🔍 问题根源
你的 **MySQL 容器** 将容器内部的 `3306` 端口映射到了宿主机的 `3307` 端口：
```
0.0.0.0:3307->3306/tcp
```
但是你的 **Spring Boot 应用配置** 中，数据库连接 URL 写的是：
```yaml
url: jdbc:mysql://mysql:3307/tlias
```
这里的 `mysql` 是容器名，在自定义网络 `wiki` 中，应用容器可以通过容器名 `mysql` 直接访问 MySQL 容器，**但这时访问的是容器内部的端口**，即 `3306`，而不是宿主机映射的 `3307`。

所以应用尝试连接 `mysql:3307`，但 MySQL 容器内部只在 `3306` 监听，连接被拒绝，导致 `Connection refused`。

---

### ✅ 解决方案
**将 application.yml 中的端口 `3307` 改为 `3306`**，即：
```yaml
spring:
  datasource:
    url: jdbc:mysql://mysql:3306/tlias   # 修改为 3306
    driver-class-name: com.mysql.cj.jdbc.Driver
    username: root
    password: 123
```

---

### 📝 操作步骤
1. **修改配置文件**（在宿主机上编辑你的 `application.yml`）。
2. **重新构建镜像**（如果配置是打包在 jar 内的）：
   ```bash
   # 停止当前容器
   docker stop tlias-server
   docker rm tlias-server
   # 重新构建镜像（假设你在项目目录）
   docker build -t tlias:1.0 .
   # 重新启动容器（注意要加入 wiki 网络）
   docker run -d --name tlias-server --network wiki -p 8080:8080 tlias:1.0
   ```
   如果配置是通过环境变量或挂载卷动态加载的，只需重启容器即可。

3. **如果不想重新构建**，也可以在启动时通过环境变量覆盖：
   ```bash
   docker run -d --name tlias-server --network wiki -p 8080:8080 \
     -e SPRING_DATASOURCE_URL="jdbc:mysql://mysql:3306/tlias" \
     tlias:1.0
   ```

---

### 🧪 验证
修改后，应用容器应该能正常连接数据库，登录请求不会再报 `Connection refused`。

---

### 💡 扩展说明
- **容器间通信**：在同一个 Docker 自定义网络中，容器之间通过 **容器名** 和 **容器内部端口** 通信，而非宿主机映射端口。宿主机的端口映射只是为了外部访问（如用 Navicat 连接数据库），容器内部互相访问时直接用容器端口。
- 你的 `nginx-tlias`、`tlias-server`、`mysql` 三个容器都在 `wiki` 网络中，所以网络是通的，只需修正端口即可。
