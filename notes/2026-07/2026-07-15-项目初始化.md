## 问题描述

需要一个有条理的方式来记录学习开发过程中遇到的问题、解决方法和感悟，并且希望能上传到 GitHub 保存和追踪。

## 解决方法

使用 Markdown 文件 + Git + GitHub 的组合：

1. **Markdown** 作为笔记格式 — GitHub 原生渲染，写什么就显示什么
2. **Git** 做版本控制 — 每次提交都是一次快照，可以回溯历史
3. **GitHub** 做云端存储 — 不会丢失，随时随地可以访问

项目结构采用「日志笔记」+「主题笔记」双模式：
- `notes/` 按日期记录，适合随手记
- `topics/` 按主题整理，适合深度总结

## 原理分析

Markdown 是纯文本格式，不依赖任何特定软件，10 年后也能打开。Git 会记录每次改动的时间、内容、说明，相当于给笔记加上了时间线。GitHub 把本地仓库同步到云端，同时提供了在线阅读和搜索功能。

## 感悟

- 好的笔记结构比好的笔记工具更重要 — 先想清楚怎么组织，再选工具
- 日志式记录降低心理门槛 — 不需要写得很完美，记下来就行
- 定期把日志笔记整理成主题笔记 — 这个过程本身就是复习和深化理解
